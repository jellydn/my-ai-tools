# Mission: 30-Day AI Learning Journey

## Why
Build a practical, evidence-backed understanding of core AI building blocks (embeddings, chunking, retrieval-augmented generation, and agentic workflows) to design and ship production-grade developer tools, specifically grounding these concepts in code taste analysis and configuration generators (like PR 309's `code-taste`).

## Success looks like
- Able to explain and visualize the impact of chunking size, overlap, and semantic boundaries on LLM retrieval accuracy.
- Build and evaluate a simple semantic search pipeline using vector embeddings and similarity ranking.
- Write customized tree-sitter or regex-based chunking rules to preserve semantic structure in codebase files (TypeScript, Markdown).
- Formulate weighted confidence equations for LLM-assisted knowledge extraction.

## Constraints
- **Time commitment**: 60-120 minutes per day.
- **Runtime**: Prefer Bun or standard Node.js for CLI tools.
- **Approach**: Hands-on builds coupled with interactive visual simulators for storage retention.

## Out of scope
- Training new foundational LLM models from scratch.
- Setting up enterprise-level production vector databases (focus on local memory-first indexers).
- In-depth study of transformer mathematical derivations (focus on practical application).

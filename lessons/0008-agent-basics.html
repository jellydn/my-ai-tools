Day 15/30 — Agent Basics

AI Agents: Observe, Act, and Reflect

How controlled loops around LLM calls enable autonomous tool usage, step budgeting, and safe stopping.

1. What is an AI Agent?

While a standard LLM completion generates text in a single pass, an AI Agent operates in a controlled, iterative loop. The agent evaluates the current goal, selects tools to execute, receives observations, and reflects on whether it has sufficient evidence to finish.

The core execution pattern follows three stages proposed in the ReAct (Reasoning + Acting) paradigm:

  1. Observe: Examine the user request, past conversation context, and latest tool results.
  2. Act: Decide whether to call a tool (e.g., list files, read code, search) or deliver the final answer to the user.
  3. Reflect: Evaluate the tool response, update internal state, and check safety bounds like remaining step budget.

2. Connecting to flue-repo-assistant

In the flue-repo-assistant codebase (agents/repo-assistant.ts), we built a bounded repository analysis agent using these exact principles:

  • Step Budgeting (createStepBudget): The agent is assigned a strict step limit (default: 8 steps). Every tool invocation consumes 1 budget unit. When budget hits 0, tool calls are blocked and the agent must answer with existing evidence.
  • Read-Only Tools: The agent interacts through constrained tools (list_files, read_file, search_code) operating inside a restricted repository sandbox.
  • Safe Stopping: Instructions require the agent to stop calling tools as soon as evidence is sufficient and fail cleanly if budget is exhausted without hallucinating missing data.
